## 4. 入库业务流程图

```mermaid
sequenceDiagram
    actor User as 操作员
    participant FE as 前端
    participant API as Flask API
    participant DB as MySQL

    User->>FE: 选择商品 + 填写数量 + 供应商
    FE->>API: POST /api/inbound/order
    Note over API: {details, supplier}
    API->>DB: INSERT inbound_order (status=pending)
    API->>DB: INSERT inbound_detail (逐条)
    DB-->>API: order_id
    API-->>FE: {id, order_no}

    FE->>API: POST /api/inbound/confirm/{id}
    API->>DB: BEGIN 事务
    API->>DB: SELECT inbound_order FOR UPDATE
    DB-->>API: 订单信息 (status=pending)

    loop 逐条明细
        API->>DB: SELECT stock FOR UPDATE
        DB-->>API: 当前库存量
        API->>DB: UPDATE stock SET quantity = old + qty
        API->>DB: INSERT stock_record
    end

    API->>DB: UPDATE inbound_order SET status=confirmed
    API->>DB: COMMIT
    API-->>FE: {order_no, message: 入库成功}
    FE-->>User: Toast: 入库成功
```

## 5. 出库业务流程图

```mermaid
sequenceDiagram
    actor User as 操作员
    participant FE as 前端
    participant API as Flask API
    participant DB as MySQL

    User->>FE: 选择商品 + 填写数量 + 客户
    FE->>FE: 前端校验库存是否充足
    Note over FE: 库存不足 → Toast 警告, 阻止提交

    FE->>API: POST /api/outbound/order
    API->>DB: INSERT outbound_order (status=pending)
    API->>DB: INSERT outbound_detail (逐条)
    DB-->>API: order_id
    API-->>FE: {id, order_no}

    FE->>API: POST /api/outbound/confirm/{id}
    API->>DB: BEGIN 事务

    loop 逐条明细 (按 product_id 升序防死锁)
        API->>DB: SELECT stock FOR UPDATE
        DB-->>API: old_qty
        alt old_qty >= qty
            API->>DB: UPDATE stock SET quantity = old - qty
            API->>DB: INSERT stock_record
        else 库存不足
            API->>DB: ROLLBACK
            API-->>FE: 400 库存不足
        end
    end

    API->>DB: UPDATE outbound_order SET status=confirmed
    API->>DB: COMMIT
    API-->>FE: {order_no, message: 出库成功}
    FE-->>User: Toast: 出库成功
```
