# 简易仓库管理系统 — 需求文档（PRD）

> 版本：v1.0  
> 日期：2026-06-06  
> 技术栈：Flask（后端）+ 原生 JS（前端）+ MySQL（数据库）

---

## 一、项目概述

### 1.1 背景与目标

本系统是一个面向中小型仓库的**简易仓库管理系统（WMS）**，支持商品管理、入库、出库、库存查询和库存流水追踪。系统采用前后端分离架构，前端为单页应用（SPA），后端提供 RESTful API。

### 1.2 核心功能

| 模块 | 功能 |
|------|------|
| 用户认证 | JWT 登录、角色区分（admin/staff） |
| 商品管理 | 商品 CRUD、编码唯一校验 |
| 入库管理 | 创建入库单、确认入库（增加库存） |
| 出库管理 | 创建出库单、确认出库（扣减库存，含库存不足校验） |
| 库存查询 | 实时库存列表、低库存预警、搜索筛选 |
| 库存流水 | 出入库变更记录、按商品/类型/日期筛选、分页 |
| 数据报表 | 近7天出入库趋势图（ECharts） |

### 1.3 技术架构

```
┌─────────────────┐      ┌─────────────────┐      ┌─────────────┐
│   浏览器前端     │  HTTP │   Flask 后端    │  TCP   │   MySQL     │
│  (SPA + 哈希路由)│  ────▶│  (Python 3.x)   │  ────▶│  (v8.0+)   │
│  Tailwind CSS   │      │  JWT + PyMySQL  │      │  warehouse  │
└─────────────────┘      └─────────────────┘      └─────────────┘
```

## 二、前端需求

### 2.1 页面结构

系统为单页应用（SPA），通过 `window.location.hash` 实现路由切换。

| 路由 | 页面 | 需登录 | 说明 |
|------|------|--------|------|
| `#login` | 登录页 | 否 | 用户名+密码登录 |
| `#dashboard` | 首页 | 是 | 统计卡片 + 趋势图 + 低库存预警 |
| `#products` | 商品管理 | 是 | 商品列表、搜索、新增/编辑/删除 |
| `#inbound` | 入库管理 | 是 | 创建入库单 + 入库单列表 |
| `#outbound` | 出库管理 | 是 | 创建出库单 + 出库单列表 |
| `#stock` | 库存查询 | 是 | 库存列表、搜索、低库存筛选 |
| `#records` | 库存流水 | 是 | 流水记录、筛选、分页 |

### 2.2 通用组件

**布局框架：**
- 侧边栏（220px，深色背景）：Logo + 导航菜单
- 顶部栏（60px）：页面标题 + 当前用户 + 退出按钮
- 主内容区：自适应宽度，白色卡片容器

**全局组件：**
- Toast 消息提示（成功/错误/警告/信息）
- 模态框（确认对话框、表单弹窗）
- 加载状态（旋转动画）
- 空状态（暂无数据图标）

### 2.3 各页面详细需求

#### 2.3.1 登录页

- 表单：用户名、密码
- 登录按钮带加载状态
- 底部显示测试账号信息
- 登录成功后存储 token + user 到 localStorage，跳转 dashboard
- 401 时自动清除 token 并跳转登录页

#### 2.3.2 首页（Dashboard）

- **4个统计卡片**：今日入库数、今日出库数、低库存预警数、商品总数
- **ECharts 柱状图**：近7天每日出入库量对比
- **低库存预警列表**：显示库存不足的商品（最多10条）

#### 2.3.3 商品管理

- 搜索框（按编码/名称模糊搜索）
- 商品表格：编码、名称、分类、单位、安全库存、创建时间、操作
- 操作：编辑、删除
- 新增按钮 → 弹窗表单（编码、名称、分类下拉、单位下拉、安全库存）
- 编辑时编码只读
- 删除前确认，删除时检查是否有关联记录

#### 2.3.4 入库管理

- **创建入库单区域**：
  - 供应商输入框（可选）
  - 动态商品明细表（选择商品 + 数量）
  - 添加/删除行按钮
  - 创建并确认入库（一步完成）
- **入库单列表**：
  - 表格：单号、状态、创建时间、确认时间、操作
  - 状态：待确认（黄色）、已确认（绿色）、已取消（红色）
  - 操作：待确认可点击确认，已确认可查看详情

#### 2.3.5 出库管理

- 与入库管理对称
- 创建出库单：客户名称（可选）+ 动态商品明细
- 出库时前端校验库存是否充足
- 库存不足时显示红色警告

#### 2.3.6 库存查询

- 搜索框（编码/名称模糊搜索）
- 仅显示低库存复选框
- 表格：编码、名称、分类、单位、当前库存、安全库存、状态、最后更新
- 低库存行红色高亮
- 库存进度条可视化

#### 2.3.7 库存流水

- 筛选：商品下拉、类型下拉（入库/出库）
- 表格：时间、编码、名称、类型、变更数量、变更前、变更后、单据号
- 分页：上一页/下一页 + 页码跳转

### 2.4 前端技术规范

- **CSS**：Tailwind CSS CDN
- **图标**：Font Awesome 6 CDN
- **图表**：ECharts 5 CDN
- **状态管理**：localStorage（token + user）
- **API 封装**：`api.js` 统一处理 baseURL、JWT 注入、401 跳转、错误提示

### 3.3 API 接口清单

| 方法 | 路径 | 说明 | 请求体 | 响应 |

#### 3.3.2 商品模块 (`/api`)

| 方法 | 路径 | 说明 | 参数 |
|------|------|------|------|
| GET | `/products` | 商品列表 | `?keyword=&category=` |

| DELETE | `/products/<id>` | 删除商品 | — |

#### 3.3.3 入库模块 (`/api/inbound`)

| 方法 | 路径 | 说明 | 请求体/参数 |
|------|------|------|------------|

#### 3.3.4 出库模块 (`/api/outbound`)

| 方法 | 路径 | 说明 | 请求体/参数 |

#### 3.3.5 库存模块 (`/api/stock`)

### 3.4 统一响应格式

```json

  "code": 200,
  "message": "success",
  "data": { ... }
}
```

错误响应：
```json
{
  "code": 400/401/403/404/405/500,
  "message": "错误描述",
  "data": {}
}
```

### 3.5 核心业务逻辑

#### 3.5.1 入库流程

```
创建入库单（pending）→ 确认入库 → 增加库存 → 记录流水
```

- 在事务中执行
- 按 product_id 升序锁行（防死锁）
- 更新 stock 表数量
- 插入 stock_record 流水记录

#### 3.5.2 出库流程

```
创建出库单（pending）→ 确认出库 → 扣减库存（校验不足）→ 记录流水
```

- 在事务中执行
- 库存不足时抛出 ValueError，回滚事务
- 前端也做初步库存校验

#### 3.5.3 库存变更核心规则

- `change_qty` 始终为正数
- `before_qty` / `after_qty` 记录变更前后的准确数量
- `biz_no` 关联对应的入库/出库单号
- `operator_id` 记录操作人

## 四、数据库设计

### 4.1 ER 图

```
user (1) ────────< (N) inbound_order
               │
               └──< (N) outbound_order
               │
               └──< (N) stock_record

product (1) ────< (N) inbound_detail
               │
               └──< (N) outbound_detail
               │
               └──< (1) stock
               │
               └──< (N) stock_record
```

### 4.2 数据表

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `user` | 用户表 | id, username, password_hash, role |
| `product` | 商品表 | id, code(唯一), name, category, unit, warn_stock |
| `inbound_order` | 入库单头 | id, order_no(唯一), supplier, operator_id, status, create_time, confirm_time |
| `inbound_detail` | 入库明细 | id, order_id, product_id, quantity |
| `outbound_order` | 出库单头 | id, order_no(唯一), customer, operator_id, status, create_time, confirm_time |
| `outbound_detail` | 出库明细 | id, order_id, product_id, quantity |
| `stock` | 库存表 | id, product_id(唯一), quantity, updated_at |
| `stock_record` | 库存流水 | id, product_id, change_type, change_qty, before_qty, after_qty, biz_no, operator_id, create_time |

### 4.3 状态枚举

- **订单状态**：`pending`（待确认）、`confirmed`（已确认）、`cancelled`（已取消）
- **角色**：`admin`（管理员）、`staff`（操作员）
- **变更类型**：`inbound`（入库）、`outbound`（出库）

## 五、环境配置

### 5.1 `.env` 配置

```
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=123456
DB_NAME=warehouse

JWT_SECRET_KEY=change-me-to-a-random-secret-string
JWT_ACCESS_TOKEN_EXPIRES=604800

FLASK_ENV=development
FLASK_DEBUG=true
```

### 5.2 启动步骤

```bash
cd backend
pip install -r requirements.txt
python init_db.py      # 初始化数据库（建表 + 测试数据）
python app.py          # 启动服务器
```

访问 `http://localhost:5000`
